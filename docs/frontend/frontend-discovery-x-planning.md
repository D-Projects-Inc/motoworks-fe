# MotoWorks: Definition, Discovery x Planning

## 1. Project Definition Phase: Purpose, Target Segment x Goals

### 1.1. Business & Brand Understanding

Before coding / designing, we need clarity:

**Business name & Offering**: “MotoWorks: Mobile Motorcycle Repairs & Maintenance”

**Core services**: 
- Engine Rebuilds
- Servicing
- Carb Tuning
- Electricals
- Diagnostics
- Custom Builds
- etc...

**Target audience**:
- Local riders (commuters, delivery drivers)
- Enthusiasts (custom & restoration)
- Emergency breakdown clients

----

### 1.2. Site Goals

| Goal                   | Purpose                            | Success Metric              |
| ---------------------- | ---------------------------------- | --------------------------- |
| Build online presence  | Show professionalism & trust       | Visitors/bookings per month |
| Simplify communication | Let clients request service easily | More WhatsApp/call leads    |
| Showcase services      | Educate & attract new riders       | Time on “Services” page     |
| Build credibility      | Display testimonials & gallery     | Reviews added monthly       |

----

### 1.3. Core Brand Feel

Using adjectives to guide design:
> Reliable • Hands-on • Gritty • Modern • Road-tested

<center>
<em>Think metal textures, mechanical icons, dark themes with orange/yellow accents.</em>
</center>

## 2. Discovery Phase: Research x Content

### 2.1. Competitor Research

- Search “motorcycle repair near me” or “bike mechanic [your city]”

- **Note**:
    - Layouts (hero sections, contact forms, maps)
    - Tone (friendly vs. technical)
    - Conversion points (buttons like “Book Now”, “WhatsApp Us”)

----

### 2.2. Information Architecture (Site Structure)

**Sitemap:**
- _Home_ - Overview, hero image, key services, CTA
- _Services_ - Detailed list with pricing tiers
- _Gallery_ / _Projects_ - Before & after photos, custom builds
- _About_ - The mechanic’s story, certifications, experience
- _Contact_ / _Booking_ - Form, WhatsApp, Map, Operating Hours
- (Optional) _Blog_ / _Tips_ - Maintenance Guides, Ride Care, etc.

----

## 3. Planning Phase: Tech x Execution

### 3.1. Tech Stack

| Stack                    | Ideal If...                            | Notes                                 |
| ------------------------ | -------------------------------------- | ------------------------------------- |
| **Astro + Bootstrap**    | You want a fast static site            | Perfect for brochure-style, PWA ready |
| **Django (with DRF)**    | You want booking system or admin panel | Can integrate with WhatsApp API       |
| **Static + Netlify CMS** | Want easy content updates              | Great for small business clients      |

----

### 3.2. Functional Features

**Essential:**
- Service list with images & brief details
- Contact/booking form + WhatsApp link
- Google Map + operating hours
- Gallery / Before & After
- Reviews/testimonials
- Customer portal (for returning clients)

_**Optional (upgrade path)**_:
- Online booking with time slots
- PWA installable version
- Payment integration

----

### 3.3. Content & Asset Checklist

| Type            | Description                               |
| --------------- | ----------------------------------------- |
| Logo            | Clean, readable even at small size        |
| Photos          | Workshop, bikes, team, before/after shots |
| Copy            | About text, service descriptions, pricing |
| Contact details | Phone, WhatsApp, email, Google Maps link  |
| Socials         | Facebook / Instagram links                |
